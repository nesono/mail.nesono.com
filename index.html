Helping Spamassassin to Learn Spam and Ham

Our mail server can learn from the messages that you tagged as Spam. It is using Spamassassin for that and it has great benefits, like your Spam detection is independent of your mail client and in it will addition reject Spam that has been detected with a specific confidence threshold.

It starts a task on a schedule that will scan all messages in your Junk folder. If your Spam folder has a different name, it will not recognize your Spam and thus not learn from it.

To learn Ham (harmless mail), you need to create a folder called Ham on the top level (next to the Inbox).

Using Spamassassin Information in Your Mail Client

Spamassassin will add headers to the mail, that you can create rules with. The following listing shows examples of the headers that Spamassassin creates.

    X-Spam-Level: ***
    X-Spam-Status: No, score=3.0 required=5.0 ...
    X-Spam-Checker-Version: SpamAssassin 3.4.1 ...

Refer to your mail client's documentation on how to create filters using mail headers.
